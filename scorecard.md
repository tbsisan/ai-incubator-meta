# Scorecard: AI Incubator Meta

## Minimal incubator rubric

### 1. What value does this create?

It creates a durable self-improvement loop for the AI Incubator. Instead of improving the system through scattered chat decisions, this meta-project can collect critiques, feature ideas, workflow gaps, and proposed parent-repo changes in one auditable place.

### 2. Who wants this, and how badly?

Tom likely wants this strongly because the incubator is becoming a recurring project-development system, not a one-off script. Demand beyond Tom is unvalidated, but similar builders may want a file-backed ideation/product-discovery agent system if it becomes easy to install and use.

### 3. Why now?

The incubator has crossed from concept into a working local system: it has idea folders, persona prompts, pass runner behavior, intake automation, model profiles, references, and git history. That makes it mature enough to review as a built project rather than only brainstorm as an idea.

### 4. What’s the smallest sharp MVP?

Create this meta-project folder and run one constrained self-review pass that produces only recommendations/proposed diffs, not direct parent-file edits. The first useful pass is probably `opportunity_scan`; `feature_brainstorm` is useful after the opportunity space is clearer.

### 5. Why would this spread?

If productized, it could spread by showing builders a concrete way to turn messy ideas into durable files, pass outputs, and implementation proposals. Its strongest shareability is practical value: “here is a reproducible local system for thinking with agents without losing state.”

### 6. Why might it fail?

It could become too complex, too recursive, too file-heavy, or too tailored to Tom’s personal workflow. It could also fail if pass outputs become verbose theater instead of concrete next actions, or if root-file changes become unsafe/unreviewed.

### 7. What’s the weird / non-obvious edge?

The system is both the workshop and a project inside the workshop. The edge is using file boundaries and promotion rules to make self-improvement possible without uncontrolled self-modification.

### 8. What should happen next?

Add a small set of mature-project pass types, then run `ai-incubator-meta` through one of them. Recommended first pass: `opportunity_scan`.
